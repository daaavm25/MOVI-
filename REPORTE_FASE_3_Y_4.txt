FASE 4 - CONTENEDORES Y FASE DE PRUEBAS
==================================================

1) OBJETIVO
Separar servicios por contenedor (backend y frontend) y ejecutar pruebas de funcionamiento, integracion, persistencia y seguridad.

2) CAMBIOS IMPLEMENTADOS
A) Backend
- Nuevo archivo Dockerfile.backend con Node 20 slim.
- Servicio backend en docker-compose.yml.

B) Frontend
- Nuevo Dockerfile en models/movie-plus-frontend usando nginx:alpine.
- Servicio frontend en docker-compose.yml.
- Puerto frontend publicado en 8080.

C) Base de datos
- Servicio db (PostgreSQL) con volumen persistente pgdata.
- Mapeo host 5433 -> contenedor 5432 para evitar conflicto local.

D) Integracion frontend-backend
- app.js y login.js ajustados para usar backend en puerto 8000 cuando el frontend corre en otro puerto.
- CORS ajustado en Compose para permitir origenes:
  http://localhost:8080,http://127.0.0.1:8080,http://localhost:8000,http://127.0.0.1:8000

E) Scripts
- docker:build actualizado a compose build.
- docker:logs actualizado para backend/frontend/db.
